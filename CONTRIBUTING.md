# 贡献指南

感谢你愿意为 **LLMGuide（大模型面试指南）** 贡献内容！无论是修正错别字、完善表述，还是新增一整个主题，都非常欢迎。本指南帮你快速上手。

## 贡献方式

- 🐛 **报告问题**：内容有错、表述不清、链接失效 → 提 [Issue](https://github.com/Meko1/llm-interview-guide/issues)。
- ✍️ **改进内容**：修正错误、补充高频追问、完善示例 → 提 Pull Request。
- 📄 **新增主题**：发现遗漏的高频考点 → 新增页面并接入导航（见下文）。
- 📚 **推荐资源**：优质课程 / 书籍 / 开源项目 → PR 到 `docs/interview/resources.md`。

## 本地开发

需要 Node.js 18+。

```bash
git clone https://github.com/Meko1/llm-interview-guide.git
cd llm-interview-guide
npm install
npm run dev        # http://localhost:5173，支持热更新
npm run build      # 提交前请确保构建通过
```

## 内容写作规范

为保持全站风格一致，新增/修改内容请遵循以下约定（参考现有页面，如 `docs/basics/transformer.md`）：

1. **面试导向、问答为主**：聚焦「面试官会怎么问」，少堆理论、少贴大段可运行代码，必要时给极简伪代码/公式即可。
2. **页面结构**：
   - 开头用一句 `> 引用` 点明本文价值与适用场景。
   - 正文用 `##` / `###` 分节，多用**对比表格**和简洁要点。
   - 结尾必须有 `## 高频追问` 小节，用 `**Q：……**` 形式列出常见追问与简明回答。
3. **图示**：本站未启用 mermaid，请用 ` ``` ` 代码块内的 ASCII 图，确保正确渲染。
4. **公式**：用 `$...$`（行内）或 `$$...$$`（独立行）书写 LaTeX。
5. **语气**：中立专业、面向通用读者；可在合适处补充「后端工程师视角」类比，但不喧宾夺主。
6. **交叉链接**：相关主题用站内链接互连，如 `[Attention](/basics/attention)`，方便读者跳转。
7. **准确性**：技术内容务必准确，拿不准的结论标注「存在争议」或给出权威出处，宁缺毋滥。

## 新增一个页面的步骤

1. 在 `docs/<板块>/` 下创建 `your-topic.md`，按上面的写作规范撰写。
2. 在 `docs/.vitepress/config.mts` 中：
   - 把页面加入对应板块的 `sidebar`；
   - 如开了新板块，记得同步更新 `nav`。
3. 如适合作为重点内容，可在 `docs/index.md` 首页的「必看」或特性卡片中露出。
4. 运行 `npm run build` 确认无报错、无死链。

## 提交 Pull Request

1. Fork 本仓库并基于 `main` 新建分支（如 `docs/add-xxx`）。
2. 提交信息建议用语义化前缀：`docs: 新增 XXX 页面` / `fix: 修正 XXX 表述` / `chore: ……`。
3. 提交前本地 `npm run build` 通过。
4. 发起 PR，简要说明改动内容与动机。
5. 维护者 Review 后合并，推送到 `main` 会自动部署上线。

## 行为准则

请保持友善、专业、尊重。我们希望这是一个对新手友好、乐于分享的开源社区。

再次感谢你的贡献！如果项目对你有帮助，也欢迎点一个 ⭐ Star。
