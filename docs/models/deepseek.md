# DeepSeek 专题（MLA / GRPO / R1）

> DeepSeek 系列以「高性价比 + 强推理」成为 2024–2025 最热门的面试话题。本文聚焦它的几项关键技术创新，几乎是当下大模型面试的必问点。

## 为什么 DeepSeek 值得单独讲？

DeepSeek 在**训练成本**和**推理性能**上都做出了有影响力的工程创新，且**开源 + 公开技术报告**，让业界第一次清楚看到「如何用更低成本训出第一梯队模型」。面试常考它的三大创新：**MLA、DeepSeekMoE、GRPO/R1 推理训练**。

## MLA：多头潜在注意力

**问题**：标准 MHA 的 KV Cache 随序列长度和头数线性增长，是长上下文推理的显存瓶颈。GQA/MQA 通过共享 KV 头来省，但会损失质量。

**MLA（Multi-head Latent Attention）的思路**：把 Key、Value **压缩成一个低维的潜在向量（latent）再缓存**，推理时通过上投影矩阵还原出各头的 K、V。

- KV Cache 只需存储这个**低秩潜在向量**，显存大幅下降（远小于 MHA，甚至小于 GQA）。
- 通过精心设计的矩阵吸收（与 RoPE 解耦处理），在省显存的同时**几乎不损失质量**——这是它优于 MQA/GQA 的关键。

> 一句话：**GQA 靠「共享 KV 头」省显存（有损），MLA 靠「低秩压缩 KV」省显存（近乎无损）**，是更激进且更优的方案。对比见 [Attention 与变体](/basics/attention)。

## DeepSeekMoE：细粒度 + 共享专家

DeepSeek 的 MoE 设计有两个特点（详见 [MoE 混合专家](/basics/moe)）：

- **细粒度专家**：把专家切得更细、数量更多，让每个专家更专业化，组合更灵活。
- **共享专家（Shared Expert）**：少数专家对所有 token 都激活，负责学习通用知识，让路由专家专注于专业分工，减少知识冗余。
- **无辅助损失的负载均衡**：用可学习的偏置项动态调整路由，避免传统辅助损失损害主任务效果。

DeepSeek-V3 总参数 671B，但每个 token 仅激活约 37B，是「大容量、低单次成本」的典型。

## 其他工程创新

- **MTP（Multi-Token Prediction，多 token 预测）**：训练时让模型一次预测多个未来 token，提供更密集的训练信号，提升数据效率；推理时还可配合做投机解码加速。
- **FP8 混合精度训练**：DeepSeek-V3 大规模采用 FP8 训练，进一步降低训练成本（详见 [分布式训练](/pretraining/distributed-training)）。

## GRPO 与 R1：用强化学习激发推理

### GRPO（Group Relative Policy Optimization）

传统 RLHF 用 PPO，需要一个与策略模型同等大小的 **Critic（价值模型）** 来估计优势，显存和成本翻倍。

**GRPO 去掉了 Critic**：对同一个问题采样**一组**回答，用**组内相对得分（相对平均水平好多少）**来代替价值函数估计优势。

$$A_i = \frac{r_i - \text{mean}(\{r_1,...,r_G\})}{\text{std}(\{r_1,...,r_G\})}$$

- 省掉价值模型 → 显存/算力大幅下降，强化学习训练更便宜。
- 特别适合**有明确可验证奖励**的任务（数学答案对错、代码能否通过测试）。

### DeepSeek-R1 训练流程

- **R1-Zero**：完全不用 SFT，直接在基座上用 GRPO 做强化学习。模型**自发涌现**出长链推理、自我反思、回溯（「aha moment」）等行为——证明了纯 RL 也能激发推理能力。代价是可读性差、语言混杂。
- **R1**：在 R1-Zero 基础上加入「冷启动 SFT 数据 + 多阶段 RL + 拒绝采样」，兼顾推理能力与可读性、通用性。
- **蒸馏（Distillation）**：用 R1 生成的高质量推理数据，去 SFT 较小的稠密模型（如 Qwen、LLaMA 系列），让小模型也获得强推理能力——蒸馏出的小模型效果甚至超过同尺寸直接做 RL 的模型。

## 高频追问

**Q：MLA 和 GQA 都是省 KV Cache，本质区别？** GQA 是让多个 Q 头**共享同一组 KV 头**（减少头数），是有损近似；MLA 是把 KV **低秩压缩成潜在向量**缓存、用时再还原，信息损失更小。MLA 省得更多且质量更好，但实现更复杂。

**Q：GRPO 相比 PPO 的最大优势？** 去掉了价值模型（Critic），用组内相对奖励估计优势，显存和计算成本几乎减半，训练更简单稳定。代价是依赖「能采样一组回答并打分」，最适合有可验证奖励的推理任务。

**Q：R1-Zero 说明了什么？** 不经过 SFT、纯靠强化学习（可验证奖励），大模型也能自发学会长链推理和自我反思。这挑战了「必须先 SFT」的认知，凸显了 RL 对推理能力的激发作用。但纯 RL 的输出可读性差，所以正式 R1 还是加了冷启动 SFT。

**Q：为什么蒸馏的小模型能超过直接 RL 的小模型？** 大模型 R1 探索出的推理模式，小模型靠自己的算力和容量很难探索到；直接把这些高质量推理轨迹「教」给小模型（蒸馏），比让小模型从零 RL 摸索更高效、效果更好。

**Q：DeepSeek 为什么便宜？** 综合多项工程优化：MLA 降推理显存、MoE 稀疏激活降单次计算、FP8 训练降训练成本、GRPO 降 RL 成本，叠加高效的训练框架工程。是「算法 + 工程」协同降本的范例。
