# 小语言模型与端侧部署（SLM）

> 不是所有场景都需要千亿大模型。小语言模型（SLM，Small Language Model）+ 端侧部署是 2024–2025 的重要趋势——更便宜、更快、更私密。这也是越来越常见的面试方向。

## 什么是 SLM？

SLM 通常指参数量在**十亿级别及以下**（约 0.5B~10B）、能在消费级硬件甚至手机上运行的语言模型。它不追求「无所不知」，而是在**特定能力或成本/延迟约束**下做到「够用且高效」。

代表模型：

| 系列 | 代表 | 特点 |
| --- | --- | --- |
| **Phi（微软）** | Phi-3 / Phi-4 | 「小而精」，靠高质量「教科书级」数据 |
| **Qwen 小尺寸** | Qwen 0.5B/1.8B/3B | 中文强、多尺寸全 |
| **Gemma（Google）** | Gemma 2B | 开源轻量 |
| **MiniCPM（面壁）** | MiniCPM 2B | 端侧友好、性能强 |
| **SmolLM（HF）** | SmolLM 135M~1.7B | 极小、可研究 |
| **Llama 小尺寸** | Llama 3.2 1B/3B | 含端侧版本 |

## 为什么需要 SLM？

| 维度 | SLM 的优势 |
| --- | --- |
| **成本** | 推理便宜得多，大规模调用省钱 |
| **延迟** | 响应快，适合实时场景 |
| **隐私** | 可本地/端侧运行，数据不出设备 |
| **离线** | 无网络也能用（手机、车机、IoT） |
| **可控** | 易私有化、易微调定制 |

> 思路转变：从「用一个超大模型干所有事」到「**用合适大小的模型干合适的事**」——简单任务用 SLM，复杂任务才路由到大模型（见 [系统设计](/engineering/system-design) 的模型分级）。

## 小模型为什么也能很强？

- **高质量数据**：Phi 系列证明，用精心筛选/合成的「教科书级」高质量数据，小模型能远超同尺寸。
- **蒸馏（Distillation）**：用大模型当老师，把能力「教」给小模型（见 [推理模型](/advanced/reasoning-models) 的 R1 蒸馏）。
- **量化**：INT4 量化后 7B 模型只需约 4GB 显存，能跑在笔记本/手机上（见 [GPU 硬件](/inference/gpu-hardware)）。
- **过度训练**：用远超 Chinchilla 最优的数据量「喂」小模型，换取部署期的高性价比（见 [缩放定律](/pretraining/scaling-law)）。

## 端侧部署

| 工具 | 平台 |
| --- | --- |
| **Ollama** | 本地一键跑开源模型（Mac/Win/Linux） |
| **llama.cpp** | C++ 推理，CPU/GPU，GGUF 量化，端侧基石 |
| **MLX** | Apple 芯片优化推理框架 |
| **MediaPipe / MLC-LLM** | 手机端（iOS/Android）部署 |

端侧关键约束：内存/算力有限 → 必须量化；功耗与发热；首 token 延迟。

## SLM vs LLM 取舍

| 维度 | SLM | LLM |
| --- | --- | --- |
| 能力广度/复杂推理 | 弱 | 强 |
| 成本/延迟 | 低 | 高 |
| 隐私/离线 | 好（可端侧） | 需云端 |
| 适合 | 明确/垂直任务、边缘、高频简单调用 | 开放复杂任务、强推理 |

## 高频追问

**Q：小模型为什么也能打？** 关键在数据质量（教科书级高质量/合成数据）、蒸馏（大模型教小模型）、以及用远超最优的数据量过度训练。Phi、MiniCPM 等证明同尺寸下「数据为王」。

**Q：端侧部署的主要约束？怎么应对？** 内存/算力/功耗受限。应对：量化（INT4/INT8 降显存）、用 llama.cpp/MLX/MLC-LLM 等优化框架、选合适尺寸的 SLM、必要时云端协同。

**Q：什么时候用 SLM 而非大模型？** 任务明确/垂直、对成本和延迟敏感、需要隐私/离线、高频简单调用时用 SLM；开放复杂任务、强推理需求用大模型。生产中常用「模型分级路由」组合两者。

**Q：SLM 和蒸馏什么关系？** 蒸馏是获得高质量 SLM 的主要手段之一：用强大模型生成高质量数据或软标签来训练小模型，让小模型以远低的成本逼近大模型在特定能力上的表现。

**Q：量化和 SLM 是一回事吗？** 不是。SLM 是「参数量本来就小」；量化是「把（任意大小）模型的权重用更少比特表示」。二者可叠加——一个 3B 的 SLM 再 INT4 量化，就能轻松跑在手机上。
