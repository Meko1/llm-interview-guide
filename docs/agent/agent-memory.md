# Agent 记忆系统

> 没有记忆的 Agent 每次都「失忆」，无法积累经验、维持长期个性化。记忆系统是 Agent 从「单次任务」走向「长期助理」的关键，也是 Agent 面试的进阶考点。

## 为什么 Agent 需要记忆？

LLM 本身是「无状态」的——每次调用只看得到当前上下文窗口里的内容，关掉就忘。但真实助理需要：

- 记住用户的偏好、历史对话（个性化）；
- 在长任务中跨多步保持状态；
- 从过往成败中积累经验。

上下文窗口是有限的「工作记忆」，装不下也不该装下所有历史 → 需要外部记忆系统。

## 记忆的分类

借鉴人类认知，Agent 记忆常分为：

| 类型 | 对应 | 说明 | 实现 |
| --- | --- | --- | --- |
| **短期记忆** | 工作记忆 | 当前任务/对话的上下文 | 上下文窗口本身 |
| **长期记忆** | 长期记忆 | 跨会话持久保存 | 外部存储（向量库/数据库） |

长期记忆又常细分为：

- **情景记忆（Episodic）**：具体发生过的事件/对话（「上次用户说他喜欢简洁回答」）。
- **语义记忆（Semantic）**：抽象的事实/知识（「用户是后端工程师」）。
- **程序记忆（Procedural）**：怎么做某事的技能/规则（常固化进 system prompt 或工具）。

## 记忆系统的工作流

```
对话/经历 ──▶ ① 写入：抽取要记的信息 ──▶ 存入长期记忆（向量库/DB）
                                              │
新一轮交互 ──▶ ② 检索：按当前需要召回相关记忆 ─┘
            ──▶ ③ 注入：把召回的记忆拼进上下文 ──▶ LLM
            ──▶ ④ 更新/遗忘：合并、修正、淘汰过时记忆
```

- **写入（Encode）**：不是把所有对话原文都存，而是**抽取**关键信息（用 LLM 总结成事实/偏好条目）。
- **检索（Retrieve）**：本质是 RAG——把记忆向量化，按当前 query 相似度召回最相关的若干条。
- **注入（Inject）**：把召回的记忆放进上下文（属于 [上下文工程](/agent/context-engineering)）。
- **更新/遗忘（Update/Forget）**：合并重复、修正过时、按时间/重要性淘汰，避免记忆库无限膨胀和自相矛盾。

## 短期记忆的管理策略

上下文窗口装不下全部历史时（详见 [LangChain 记忆](/engineering/langchain)）：

- **全量缓冲**：留全部历史，简单但很快超窗。
- **滑动窗口**：只留最近 N 轮。
- **摘要**：把旧对话压成摘要。
- **摘要 + 窗口混合**：近的留原文、远的留摘要。

## 主流实现

- **向量库记忆**：把记忆条目 Embedding 后存向量库，按相似度检索（最通用）。
- **专用记忆框架**：如 **Mem0**、**Letta（原 MemGPT）**、Zep 等，提供记忆抽取、分层、更新的开箱方案。
- **MemGPT 思想**：把 LLM 当「操作系统」，在「主上下文（内存）」和「外部存储（磁盘）」间像分页一样换入换出信息，突破窗口限制。

## 高频追问

**Q：Agent 的短期记忆和长期记忆有什么区别？** 短期记忆 = 当前上下文窗口（工作记忆，会话结束即失效）；长期记忆 = 存在外部（向量库/DB）的持久信息，跨会话可检索回来。短期受窗口大小限制，长期理论上无限但需检索。

**Q：长期记忆本质上是什么技术？** 本质是 RAG：把记忆条目向量化存储，用到时按相关性检索回来注入上下文。区别在于记忆还需要「写入时抽取/总结」和「更新/遗忘」机制，而不只是被动检索。

**Q：记忆为什么不能把对话原文全存？** 会爆炸式增长、检索噪声大、还可能存进矛盾信息。应该用 LLM 抽取关键事实/偏好成结构化条目，并定期合并、修正、淘汰，保持记忆库精炼一致。

**Q：情景记忆和语义记忆的区别？** 情景记忆是「具体发生的事」（某次对话、某个事件）；语义记忆是「抽象的事实/知识」（用户的身份、长期偏好）。前者偏原始记录，后者偏提炼结论，检索和使用方式不同。

**Q：MemGPT 的核心思想？** 把 LLM 类比操作系统：上下文窗口是「内存」，外部存储是「磁盘」，通过自主地在两者间换入换出信息（分页），让 Agent 在有限窗口下管理远超窗口的长期记忆。
