# 视频生成

> Sora、可灵、Runway 把文生视频推到了风口。视频生成在图像生成（[扩散模型](/multimodal/diffusion)）基础上多了「时间维度」，是多模态生成的前沿，也是越来越常见的面试谈资。

## 一、视频生成难在哪？

视频 = 图像 + 时间。相比文生图，文生视频要额外解决：

- **时序一致性（Temporal Consistency）**：相邻帧要连贯，物体不能忽大忽小、闪烁、变形。
- **运动合理性**：符合物理规律的运动（重力、碰撞、流体）。
- **长时程**：生成几秒甚至更长，且全程保持一致。
- **巨大计算量**：多帧 × 高分辨率，token/像素数暴涨，训练和推理都极贵。

## 二、主流技术路线

### 扩散 + 时空建模

主流做法是把[扩散模型](/multimodal/diffusion)从空间扩展到**时空**：在「帧序列」上做去噪，并用时间维度的注意力/卷积建模帧间关系。

- 早期：在 U-Net 里加**时间注意力层**（Temporal Attention），让帧之间交互。
- 现在：**DiT（Diffusion Transformer）+ 时空 patch**。

### Sora 的核心：时空 patch（Spacetime Patches）

Sora 的关键思想：把视频切成**时空块（spacetime patches）**——类似 ViT 把图像切 patch，但 patch 跨越了空间和时间。这些时空 patch 作为 token 输入 DiT 做扩散去噪。好处：

- 统一处理不同分辨率、时长、宽高比的视频/图像。
- 像 LLM 处理 token 一样「可扩展」——这也是 Sora 强调的「视频生成的 scaling」。

### 先压缩再生成

和 Latent Diffusion 一样，先用一个视频编解码器把视频压到**低维潜空间**（在时空上都压缩），在潜空间做扩散，再解码回像素，大幅降算力。

## 三、控制与一致性技术

- **文本控制**：用文本编码器 + 交叉注意力注入 prompt（同 [扩散模型](/multimodal/diffusion)）。
- **图生视频（I2V）**：给首帧图，生成后续运动。
- **运动/相机控制**：指定镜头运动、轨迹。
- **一致性**：参考帧、关键帧插值、更长的时序注意力窗口。

## 四、代表产品

- **Sora（OpenAI）**：DiT + 时空 patch，长时高质。
- **可灵（快手）、即梦（字节）**：国产文生视频代表。
- **Runway Gen-3、Pika、Luma**：海外创意视频工具。
- **Stable Video Diffusion、Wan、HunyuanVideo**：开源视频生成模型。

## 五、与世界模型的关系

视频生成被认为可能通向**世界模型（World Model）**——如果模型能准确预测「下一帧会发生什么」，它就隐含地学到了物理规律和世界运行方式。Sora 被 OpenAI 定位为「世界模拟器」的雏形，这是视频生成更深远的意义（但目前仍会违反物理、出现穿模等错误）。

## 六、高频追问

**Q：视频生成和图像生成的主要区别？** 多了时间维度，要解决时序一致性（帧间连贯）、运动合理性、长时程一致和巨大计算量。技术上是把扩散模型从空间扩展到时空（时空注意力 / 时空 patch）。

**Q：Sora 的核心技术思想？** 把视频切成跨越空间和时间的「时空 patch」作为 token，用 DiT（Diffusion Transformer）在潜空间做扩散去噪。这统一了不同分辨率/时长/宽高比的处理，并让视频生成像 LLM 一样可扩展。

**Q：为什么视频生成要先压缩到潜空间？** 视频的像素数 = 帧数 × 分辨率，直接在像素空间做扩散计算量爆炸。先用视频编解码器在时空上压缩到低维潜空间，在潜空间扩散再解码，大幅降低算力，和 Stable Diffusion 的 Latent Diffusion 同理。

**Q：怎么保证帧间一致性？** 用时间维度的注意力/卷积建模帧间关系、时空联合建模、参考帧/关键帧约束、更长的时序窗口；从架构上让模型「看到」前后帧而非独立生成每帧。

**Q：为什么说视频生成可能是「世界模型」？** 若模型能准确预测下一帧，就隐含学到了物理规律和世界动态。能模拟世界的生成模型可作为智能体的「想象/规划」环境。但当前模型仍常违反物理（穿模、物体凭空出现），离真正的世界模型还有距离。
