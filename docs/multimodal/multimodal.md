# 多模态大模型

> 多模态让模型能同时理解文本、图像、音频等。本文聚焦视觉-语言模型（VLM）的核心机制：从 CLIP、ViT 到 LLaVA 架构、训练范式、原生多模态与高分辨率处理。图像生成见 [扩散模型](/multimodal/diffusion)，语音见 [语音大模型](/multimodal/speech)。

## 一、什么是多模态大模型？

多模态大模型（Multimodal LLM / VLM）能处理多种模态的输入（甚至输出），最常见是「图文」结合——既能看图又能用语言推理，完成看图问答（VQA）、OCR、图表理解、图像描述、文档理解等任务。代表：GPT-4o、Qwen-VL、LLaVA、Gemini、InternVL。

**核心挑战**：如何把**图像信息对齐到语言模型能理解的表示空间**——让 LLM「看懂」图。

## 二、CLIP：图文对比学习的基石

CLIP（Contrastive Language-Image Pre-training）用海量「图-文」对做**对比学习**：

- 在一个 batch 内，让 N 个**匹配**的图文对在向量空间靠近、N²−N 个**不匹配**对远离（对称的对比损失）。
- 训练出的图像编码器和文本编码器**共享同一语义空间**。

意义：实现强大的**零样本图像分类**和图文检索；其图像编码器（ViT）被广泛用作多模态模型的**视觉骨干**。

## 三、ViT：把图像变成 token

ViT（Vision Transformer）把图像切成固定大小的小块（patch，如 16×16），每个 patch 线性映射成向量（相当于图像的「token」），加位置编码后送入标准 Transformer。这让图像能用与文本统一的方式被建模，是视觉编码器的主流。

## 四、主流架构：以 LLaVA 为例

典型 VLM 由三部分组成：

```
   图像 ──▶ ┌──────────────┐  视觉特征  ┌──────────┐ 视觉 token
            │ 视觉编码器     │ ────────▶ │ 投影层    │ ─────────┐
            │ (CLIP ViT)    │           │ MLP/Q-Fmr│          │
            └──────────────┘           └──────────┘          ▼
                                                       ┌─────────────┐
   文本 ─────────────── 文本 token ──────────────────▶│     LLM      │──▶ 回答
                                                       └─────────────┘
```

1. **视觉编码器（Vision Encoder）**：通常是 CLIP 的 ViT，把图像编码成视觉特征。
2. **连接器 / 投影层（Projector）**：把视觉特征映射到 LLM 的词嵌入空间。LLaVA 用简单的 MLP；BLIP-2 用 **Q-Former**（一组可学习 query 提取并压缩视觉信息）。
3. **大语言模型（LLM）**：把投影后的「视觉 token」和文本 token 拼接，统一处理并生成回答。

**训练两阶段**：先用图文对训练**对齐**（通常冻结视觉编码器和 LLM、只训投影层），再用多模态指令数据做**指令微调**。

## 五、原生多模态 vs 适配器式

- **适配器式（如早期 LLaVA）**：在已训练好的 LLM 上接视觉编码器 + 投影层，后期对齐。成本低、灵活，但融合较浅。
- **原生多模态（如 Gemini 2.5、GPT-4o/GPT-5）**：从**预训练阶段**就混合多种模态数据一起训练，模态融合更深、能力更强（也更能做 any-to-any），但训练成本高。

> 2025 趋势：主流旗舰模型（GPT-5、Gemini 2.5、Llama 4、Claude 4）全部走向原生多模态。适配器式路线仍存在于开源中（如 Qwen2-VL、InternVL 2.5），且通过更好的连接器设计（如像素 shuffle、多分辨率动态切分）持续缩小差距。

## 六、工程挑战：高分辨率与 token 爆炸

高清图切成 patch 后视觉 token 很多，挤占上下文、变慢。常见对策：

- **切图分块**分别编码（如 LLaVA-NeXT 的 AnyRes）。
- **Q-Former / Resampler** 压缩 token 数。
- **动态分辨率**（Qwen-VL 按图大小调整 token 数）。

## 七、多模态幻觉

VLM 也有幻觉，称为**物体幻觉**——描述图中不存在的物体。成因：视觉对齐不充分、**语言先验过强**（凭常识猜测而非真看图）。缓解：更强的视觉对齐、更高分辨率、针对性评测（如 POPE）。

## 八、高频追问

**Q：图像信息是怎么"喂"给 LLM 的？** 视觉编码器把图像变成特征，投影层把特征映射成与文本 token 同维度的「视觉 token」，与文本 token 拼在一起输入 LLM，模型把它们一视同仁地处理。

**Q：CLIP 的训练目标是什么？** 对比学习：在一个 batch 内让 N 个正确图文对相似度最大化、N²−N 个错误配对相似度最小化（对称对比损失），使图像和文本编码器共享同一语义空间。

**Q：为什么需要 Q-Former 这类连接器？** 视觉特征 token 数多、与语言空间不对齐。Q-Former 用少量可学习 query 提取并压缩最相关的视觉信息，减少 token 数并完成模态对齐，降低 LLM 负担。

**Q：典型 VLM 的训练分几步？** 两步：① 对齐预训练——用图文对训练投影层（常冻结视觉编码器和 LLM），让视觉特征对齐到语言空间；② 指令微调——用多模态指令数据让模型学会按指令完成视觉任务。

**Q：原生多模态和适配器式的区别？** 适配器式在现成 LLM 上后接视觉模块、后期对齐，融合浅、成本低；原生多模态从预训练就混合多模态数据，融合深、能力强（更利于 any-to-any），但训练成本高。

**Q：多模态模型也有幻觉吗？** 有，主要是「物体幻觉」——描述图中不存在的物体。成因是视觉对齐不充分、语言先验过强（凭常识猜而非真看图）。

**Q：高分辨率大图怎么处理？token 会不会爆炸？** 会。对策：切图分块分别编码（AnyRes）、用 Q-Former/Resampler 压缩 token、动态分辨率（按图大小调 token 数）。

**Q：CLIP 这类对比模型和生成式 VLM 有什么不同？** CLIP 是判别/检索模型，把图文映射到同一空间算相似度，不能生成文本；生成式 VLM（LLaVA/Qwen-VL）借用 CLIP 的视觉编码器，接上 LLM 后能基于图像生成自由文本。CLIP 常作为后者的视觉骨干。

**Q：除了图像，多模态还包括什么？** 音频（[语音](/multimodal/speech)）、视频（[视频生成](/multimodal/video-generation)，时序帧+音轨）、3D 等。前沿是「any-to-any」——任意模态输入、任意模态输出（如 GPT-4o 的实时语音、图像生成）。
