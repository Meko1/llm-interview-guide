# 多模态大模型

> 多模态让模型能同时理解文本、图像、音频等。本文聚焦视觉-语言模型（VLM）的核心机制。

## 什么是多模态大模型？

多模态大模型（Multimodal LLM / VLM）能处理多种模态的输入（甚至输出），最常见是「图文」结合——既能看图又能用语言推理，完成看图问答、OCR、图表理解、图像描述等任务。代表：GPT-4V、Qwen-VL、LLaVA、Gemini。

核心挑战：如何把**图像信息对齐到语言模型能理解的表示空间**。

## CLIP：图文对比学习的基石

CLIP（Contrastive Language-Image Pre-training）用海量「图-文」对做**对比学习**：让匹配的图文对在向量空间中靠近、不匹配的远离。训练出的图像编码器和文本编码器共享同一语义空间。

- 意义：实现了强大的**零样本图像分类**和图文检索；它的图像编码器（如 ViT）被广泛用作多模态模型的视觉骨干。

## ViT：把图像变成 token

ViT（Vision Transformer）把图像切成固定大小的小块（patch），每个 patch 线性映射成向量（相当于图像的「token」），加上位置编码后送入标准 Transformer。这让图像能用与文本统一的方式被建模，是视觉编码器的主流。

## 主流架构：以 LLaVA 为例

典型 VLM 由三部分组成：

1. **视觉编码器（Vision Encoder）**：通常是 CLIP 的 ViT，把图像编码成视觉特征。
2. **连接器 / 投影层（Projector）**：把视觉特征映射到 LLM 的词嵌入空间。LLaVA 用简单的 MLP；BLIP-2 用 **Q-Former**（一组可学习 query 提取并压缩视觉信息）。
3. **大语言模型（LLM）**：把投影后的「视觉 token」和文本 token 拼接，统一处理并生成回答。

训练通常两阶段：先用图文对训练**对齐**（只训投影层），再用多模态指令数据做**指令微调**。

## 高频追问

**Q：图像信息是怎么"喂"给 LLM 的？** 视觉编码器把图像变成特征，投影层把这些特征映射成和文本 token 同维度的「视觉 token」，与文本 token 拼在一起输入 LLM，模型把它们一视同仁地处理。

**Q：CLIP 的训练目标是什么？** 对比学习：在一个 batch 内，让 N 个正确图文对的相似度最大化、N²−N 个错误配对的相似度最小化（对称的对比损失）。

**Q：为什么需要 Q-Former 这类连接器？** 视觉特征 token 数多、与语言空间不对齐。Q-Former 用少量可学习 query 提取并压缩最相关的视觉信息，减少 token 数并完成模态对齐，降低 LLM 负担。

**Q：多模态模型也有幻觉吗？** 有，称为「物体幻觉」——描述图中不存在的物体。成因包括视觉对齐不充分、语言先验过强（凭常识猜测而非真看图）。
