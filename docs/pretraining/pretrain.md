# 预训练目标与数据

> 预训练是大模型能力的根基。本文讲清训练目标、数据处理与训练范式的高频考点。

## 预训练目标

不同架构对应不同的自监督训练目标：

| 目标 | 全称 | 做法 | 代表 |
| --- | --- | --- | --- |
| CLM | Causal LM（因果语言建模） | 预测下一个 token，单向 | GPT、LLaMA |
| MLM | Masked LM（掩码语言建模） | 随机遮盖 15% token 让模型还原，双向 | BERT |
| PrefixLM / Span Corruption | 前缀语言建模 / 片段还原 | 还原被遮盖的连续片段 | T5、GLM |

**主流 LLM 用 CLM（下一个 token 预测）**，因为：训练目标统一、能利用全部文本、天然适配生成与对话、规模扩展表现好。

损失函数是交叉熵：让模型预测的下一个 token 概率分布尽可能接近真实的下一个 token。

## 训练数据

数据质量往往比数量更决定模型上限，「Garbage in, garbage out」。典型来源包括网页（Common Crawl）、书籍、代码（GitHub）、百科、论文、问答社区等。

数据处理流水线（高频考点）：

1. **清洗**：去除 HTML 标签、乱码、过短/过长文本、低质量页面。
2. **去重**：文档级 + 句子级去重（如 MinHash / SimHash），去重对效果至关重要，能防止记忆与过拟合。
3. **质量过滤**：用规则或分类器筛掉低质内容（如用一个分类器判断是否「像维基百科」）。
4. **去毒/隐私**：过滤有害内容、PII 个人隐私。
5. **配比与混合**：调整不同来源（代码、中英文、数学）的比例，配比直接影响能力分布。
6. **Tokenize 与打包**：切成 token 后拼接成定长序列。

## 训练范式与并行

大模型训练需要分布式并行：

- **数据并行（DP）**：每张卡放完整模型，喂不同数据，再同步梯度。
- **张量并行（TP）**：把单层的矩阵切到多张卡上算（层内拆分）。
- **流水线并行（PP）**：把不同层放到不同卡上，像流水线一样传递（层间拆分）。
- **ZeRO（DeepSpeed）/ FSDP**：把优化器状态、梯度、参数分片到多卡，大幅降低单卡显存。

实际超大模型训练通常是这几种并行的组合（3D 并行）。

混合精度（FP16/BF16）训练 + 梯度累积是标配；**BF16** 因动态范围大、不易溢出，成为大模型训练首选。

## 高频追问

**Q：BERT（MLM）和 GPT（CLM）的本质区别？** BERT 双向、看得到全文，擅长理解类任务但不能直接生成；GPT 单向、只能看前文，天然适配自回归生成。

**Q：为什么去重这么重要？** 重复数据会让模型过度记忆、降低泛化，还会浪费算力；研究表明充分去重能显著提升下游效果。

**Q：预训练和微调的数据量差多少？** 预训练是万亿（trillion）token 量级，微调（SFT）通常是几万到几百万条高质量样本，差好几个数量级。

**Q：什么是 continue pre-training（继续预训练）？** 在已有基座上用特定领域（如医疗、法律、代码）语料继续做 CLM 训练，注入领域知识，再做 SFT，常用于行业大模型。
