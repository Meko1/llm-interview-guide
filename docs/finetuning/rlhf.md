# RLHF / DPO 对齐

> 对齐让模型输出更符合人类偏好（有用、诚实、无害）。RLHF 与 DPO 是高频考点。

## 为什么需要对齐？

SFT 后的模型已经会遵循指令，但不一定输出**人类最偏好**的回答（可能啰嗦、不安全、不够有用）。对齐阶段进一步用「人类偏好」信号优化模型，使其行为符合 **3H 原则：Helpful（有用）、Honest（诚实）、Harmless（无害）**。

## RLHF 三阶段

RLHF（Reinforcement Learning from Human Feedback，基于人类反馈的强化学习）经典流程：

1. **SFT**：先得到一个会遵循指令的基础模型（见 [微调范式](/finetuning/finetuning)）。
2. **训练奖励模型（Reward Model, RM）**：
   - 对同一 prompt，让模型生成多个回答，由人类标注「哪个更好」（排序/两两比较，而非打绝对分，因为相对判断更可靠）。
   - 用这些偏好数据训练一个 RM，输入「prompt+回答」，输出一个标量分数，拟合人类偏好。
3. **强化学习优化（通常用 PPO）**：
   - 把语言模型当作策略（policy），生成回答；
   - 用 RM 给回答打分作为奖励；
   - 用 PPO 更新模型，让它倾向于产生高奖励的回答；
   - 加上 **KL 散度惩罚**，约束新模型不要偏离 SFT 模型太远，防止「奖励欺骗」（reward hacking）和能力退化。

## PPO 为什么复杂？

PPO（Proximal Policy Optimization）训练时需要同时加载 **4 个模型**：策略模型、参考模型（算 KL）、奖励模型、价值模型（critic）。显存和工程复杂度高、训练不稳定、调参困难——这正是 DPO 出现的动机。

## DPO（Direct Preference Optimization）

DPO 的核心洞见：**可以跳过显式训练奖励模型和 RL，直接用偏好数据优化策略**。它通过数学推导，把 RLHF 的目标转化为一个简单的分类式损失，直接在 `(prompt, 好回答 y_w, 坏回答 y_l)` 三元组上做监督训练。

- 优点：不需要单独的 RM 和 RL 循环，只需 2 个模型（策略 + 参考），稳定、简单、省资源，效果常可比肩 PPO。
- 已成为开源社区主流对齐方法。

## 其他对齐方法

- **RLAIF**：用 AI（更强的模型）代替人类标注偏好，降低标注成本（如 Constitutional AI）。
- **KTO**：只需「好/坏」二元标签，无需成对比较，数据更易获取。
- **GRPO**：DeepSeek 在推理模型训练中使用，去掉价值模型、用组内相对奖励，进一步简化 RL。
- **拒绝采样（Rejection Sampling）**：生成多个回答，用 RM 选最好的加入 SFT 数据再训练。

## 高频追问

**Q：RLHF 和 SFT 的区别？** SFT 是「模仿」标准答案（监督学习）；RLHF 是基于「比较/打分」优化，能学到比示范更好的策略，也能压制不想要的行为。

**Q：DPO 相比 PPO 的优劣？** DPO 简单稳定省资源，工程友好；但 PPO 的在线探索在某些场景（尤其需要在线生成反馈、复杂奖励）上仍有优势。DPO 是离线的，依赖固定的偏好数据集。

**Q：KL 惩罚的作用？** 防止模型为了迎合奖励模型而过度偏离原始分布，导致语言能力退化或输出怪异（reward hacking）。它是约束「对齐税」的关键。

**Q：什么是「对齐税（Alignment Tax）」？** 对齐过程可能让模型在某些基准能力上略有下降，是对齐安全/偏好与原始能力之间的权衡。
