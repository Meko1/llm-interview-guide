# LoRA / QLoRA 详解

> LoRA 是目前最主流的参数高效微调方法，几乎是面试必考。本文讲清原理、超参数与 QLoRA。

## LoRA 原理

LoRA（Low-Rank Adaptation，低秩适配）的核心假设：**微调时权重的变化量 ΔW 是低秩的**，可以用两个小矩阵的乘积来近似。

对一个原始权重矩阵 W（维度 d×d），不直接更新它，而是冻结 W，并在旁边加一个低秩旁路：

$$h = Wx + \Delta W x = Wx + BA x$$

其中 A 是 d×r、B 是 r×d 的矩阵，秩 `r ≪ d`（如 r=8、16）。训练时只更新 A、B，参数量从 d² 降到 2dr，可能只有原来的千分之几。

- 初始化：A 用随机高斯，B 初始化为 0，保证训练开始时 ΔW=0，不破坏原模型。
- 缩放：实际用 `ΔW = (α/r)·BA`，α 是缩放超参。

## LoRA 的优点

- **省显存省算力**：只训练极少参数，单张消费级显卡也能微调大模型。
- **无推理延迟**：训练完可以把 BA 合并回 W（`W' = W + BA`），推理时与原模型结构完全一致，零额外开销。
- **多任务切换灵活**：基座只存一份，每个任务只存一个几 MB 的 LoRA 权重，按需加载/热插拔。

## 关键超参数

| 参数 | 含义 | 经验 |
| --- | --- | --- |
| r（秩） | 旁路矩阵的秩 | 8/16/32/64，越大容量越强也越易过拟合 |
| α（alpha） | 缩放系数 | 常设为 r 的 1～2 倍 |
| target_modules | 应用 LoRA 的层 | 至少 q_proj、v_proj；加上 k/o/FFN 效果更好 |
| dropout | LoRA 旁路 dropout | 0.05 左右防过拟合 |

## QLoRA

QLoRA = **量化（Quantization）+ LoRA**，让在单张 GPU 上微调超大模型成为可能。三个关键技术：

1. **4-bit NF4 量化**：把冻结的基座权重量化到 4-bit（NormalFloat4，针对正态分布权重优化），大幅降低显存。
2. **双重量化（Double Quantization）**：连量化常数本身也量化，进一步省显存。
3. **分页优化器（Paged Optimizers）**：用 NVIDIA 统一内存，防止显存峰值时 OOM。

效果：基座以 4-bit 存储（前向时反量化为 BF16 计算），梯度只流经 16-bit 的 LoRA 旁路，从而在保持接近全参微调效果的同时，把 65B 模型的微调显存压到单张 48GB 卡可承受。

## 高频追问

**Q：LoRA 为什么不增加推理延迟，而 Adapter 会？** LoRA 的旁路可以在推理前数学上合并进原权重（`W+BA`），结构不变；Adapter 是串行插入的新模块，推理时必须额外前向，增加延迟。

**Q：r 设多大合适？** 取决于任务复杂度和数据量。简单任务 r=8 足够；复杂任务或数据多可加大。r 过大会增加过拟合风险和存储，收益递减。

**Q：QLoRA 量化的是哪部分？会掉精度吗？** 量化的是**冻结的基座权重**，LoRA 旁路仍是高精度。NF4 + 反量化计算把精度损失控制得很小，效果接近 16-bit LoRA。

**Q：LoRA 能注入新知识吗？** 能力有限。LoRA 更擅长调整风格、格式、任务适配；要可靠注入大量新事实，继续预训练或 RAG 更合适。
