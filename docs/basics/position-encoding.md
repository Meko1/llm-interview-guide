# 位置编码（Positional Encoding）

> Attention 本身对顺序无感知，必须显式注入位置信息。位置编码是长上下文相关面试题的核心。

## 为什么需要位置编码？

自注意力的计算是「置换不变」的——打乱输入 token 的顺序，输出只是相应地打乱，模型无法区分「我打你」和「你打我」。因此必须额外提供位置信息。

## 主要方案

### 1. 正弦绝对位置编码（Sinusoidal，原始 Transformer）

用不同频率的 sin/cos 函数为每个位置生成固定向量，加到词嵌入上。

- 优点：无需训练参数；理论上可外推到训练时未见过的长度。
- 缺点：是绝对位置，表达相对关系的能力弱；实际外推效果一般。

### 2. 可学习绝对位置编码（Learned，BERT/GPT）

把位置当作可学习的 Embedding。简单有效，但**无法外推**到超过训练长度的位置（没见过的位置没有对应向量）。

### 3. 旋转位置编码（RoPE，主流）

RoPE（Rotary Position Embedding）是当前大模型（LLaMA、Qwen、GLM 等）的主流方案。核心思想：**通过对 Q、K 向量按位置「旋转」一个角度，把绝对位置信息编码进去，而它们的点积只依赖相对位置**。

- 优点：同时具备绝对与相对位置的优点；作用在 Attention 内部、不增加参数；外推性较好，且便于通过插值扩展上下文。
- 这也是为什么很多长上下文扩展方案（如 NTK-aware、位置插值 PI、YaRN）都是围绕 RoPE 做文章。

### 4. ALiBi（Attention with Linear Biases）

不加位置向量，而是直接给注意力分数加上一个与「相对距离成正比」的线性惩罚（距离越远惩罚越大）。

- 优点：实现简单，外推能力强（在远超训练长度时仍能工作）。
- 代表：BLOOM、MPT。

## 长上下文扩展方法

把一个 4K 上下文的模型扩展到 32K/128K，常见思路（多基于 RoPE）：

- **位置插值（PI, Position Interpolation）**：把超出范围的位置「压缩」回训练范围内，需少量微调。
- **NTK-aware Scaling**：调整 RoPE 的频率基（base），高频维度少缩放、低频维度多缩放，几乎不需微调即可外推。
- **YaRN**：结合 NTK 与温度调整，是目前效果较好的长上下文扩展方法之一。

## 高频追问

**Q：RoPE 为什么能体现相对位置？** 对位置 m 和 n 的向量分别旋转 mθ、nθ 后做点积，结果只与角度差 (m−n)θ 有关，从而把相对位置编码进了注意力分数。

**Q：为什么可学习位置编码不能外推？** 它为每个绝对位置单独学一个向量，训练时没见过的位置（如第 5000 位）没有对应的、有意义的向量。

**Q：长上下文的主要挑战是什么？** 一是位置编码的外推；二是 KV Cache 显存随长度线性增长；三是注意力 O(n²) 的计算开销；四是「大海捞针」式的有效信息检索能力（中间内容易被忽略，即 lost in the middle）。
