# 模型评估与幻觉

> 「怎么衡量一个大模型好不好」以及「幻觉如何产生与缓解」是高频考点。

## 评估方法分类

### 1. 自动指标（基于参考答案）

- **困惑度（Perplexity, PPL）**：衡量语言模型对测试集的预测能力，越低越好。反映语言建模能力，但与下游任务表现不完全一致。
- **BLEU**：机器翻译常用，基于 n-gram 精确率（生成与参考的重合度）。
- **ROUGE**：摘要常用，偏召回率（参考内容被覆盖的程度）。
- **BERTScore**：用语义向量计算相似度，比 n-gram 更贴近语义。

这些指标对开放式生成不够准（同义不同词会被判低分）。

### 2. 基准测试（Benchmark）

用标准化数据集评测特定能力：

- **MMLU**：多学科知识与理解。
- **GSM8K / MATH**：数学推理。
- **HumanEval / MBPP**：代码生成。
- **C-Eval / CMMLU**：中文能力。
- **BBH**：高难度推理。

注意**数据污染（Data Contamination）**：测试集泄漏进训练数据会虚高分数，是评测可信度的大问题。

### 3. 人类评估

人工对回答打分或排序，最贴近真实偏好，但贵、慢、主观。**Chatbot Arena（LMSYS）** 用真实用户两两盲评 + Elo 评分，是公认权威的人类偏好排行。

### 4. LLM-as-a-Judge

用更强的 LLM（如 GPT-4）当裁判给回答打分或比较。优点是可扩展、接近人类判断；缺点是有偏差（如**位置偏好**——偏向靠前的答案、**长度偏好**——偏好更长回答、**自我偏好**——偏向同源模型）。需要通过随机交换位置、设计好评分 rubric 来缓解。

## 幻觉（Hallucination）

幻觉指模型生成**看似合理但实际错误或无依据**的内容。分为：

- **事实性幻觉**：与客观事实不符（编造数据、人物、引用）。
- **忠实性幻觉**：与给定上下文/指令不一致（如 RAG 中答案脱离检索材料）。

### 成因

- 训练数据存在错误、过时或覆盖不足；
- 模型本质是概率生成，倾向于「说得流畅」而非「确保正确」；
- 知识截止 + 缺乏实时信息；
- 对齐时奖励了「自信回答」，模型不愿说「我不知道」；
- 长尾知识记忆不牢、压缩有损。

### 缓解手段

- **RAG**：基于检索材料作答并附出处（最有效的事实性缓解手段之一）。
- **提示工程**：要求模型「不确定就说不知道」、先给依据再下结论、CoT 推理。
- **解码控制**：降低温度、约束生成。
- **事实校验 / 后处理**：用工具或二次模型核查、引用比对。
- **训练层面**：高质量数据、RLHF 中奖励诚实与「拒答」、教模型校准置信度。
- **自一致性 / 自我反思**：多次采样比对、让模型复查自己的答案。

## 高频追问

**Q：困惑度低就一定好吗？** 不一定。PPL 衡量语言建模能力，但不直接等于指令遵循、推理、安全性等下游能力，需结合任务基准。

**Q：为什么模型不愿说"不知道"？** 训练数据中"自信回答"占多数，且对齐时正确回答被奖励、拒答有时被视为不够有用，导致模型倾向于猜测而非承认无知。可通过专门的拒答/校准训练缓解。

**Q：幻觉能彻底消除吗？** 难以根除，因为它源于生成式模型的概率本质。目标是显著降低并可控（溯源、可校验），而非保证 100% 正确。

**Q：怎么评估 RAG 系统？** 见 [RAG 进阶](/rag/rag-advanced)：分检索（命中率、上下文相关性）与生成（忠实度、答案相关性）两部分，常用 RAGAS。
