# 结构化输出详解

> 让大模型稳定输出程序能解析的格式（JSON、枚举、特定 Schema）是 AI 工程落地的刚需——Agent 工具调用、数据抽取、前后端对接都依赖它。「怎么保证模型一定输出合法 JSON」是高频工程题。

## 为什么结构化输出是难点？

LLM 本质是「文字接龙」，天生输出的是自由文本。直接要求它输出 JSON 时常见翻车：

- 多了解释性文字（「好的，这是你要的 JSON：……」）；
- 用了 Markdown 代码块包裹（```json）；
- 引号/逗号/括号不匹配，JSON 非法；
- 字段缺失、类型不对、枚举值乱编。

工程上需要从「请求」到「解码」多层保障来根治。

## 实现手段（从弱到强）

| 手段 | 原理 | 可靠性 |
| --- | --- | --- |
| **Prompt 约束** | 在提示里要求输出 JSON 并给示例 | 最弱，模型可能不守规矩 |
| **JSON Mode** | API 保证输出是**合法 JSON**（但不保证符合你的 schema） | 中 |
| **Schema 约束 / 约束解码** | 传入 JSON Schema，**在解码层强制**每个 token 都符合结构 | 最强，几乎 100% 合规 |
| **重试 + 校验** | 输出后用 schema 校验，失败把错误回喂重试 | 兜底补充 |

### 约束解码（Constrained Decoding）原理

最可靠的方式。核心思想：**在生成每个 token 时，根据目标格式（Schema/正则/语法）屏蔽掉所有「不合法」的候选 token**，只允许从合法集合里采样。

- 例如已经生成了 `{"age":`，下一步只允许数字相关 token，不允许字母。
- 通过把 JSON Schema 编译成**有限状态机/语法**来约束解码路径，保证产出一定能解析、且符合结构。
- 代表实现：OpenAI **Structured Outputs**、**Outlines**、**XGrammar**、vLLM/TGI 的 **guided decoding**、llama.cpp 的 GBNF 语法。

> 区别记牢：**JSON Mode 保证「是合法 JSON」，Schema 约束保证「符合你定义的 JSON 结构」**，后者更强。

## 工程最佳实践

- **优先用原生能力**：能用 API 的 Structured Outputs / Schema 约束就别只靠 prompt。
- **Schema 设计要简单**：嵌套别太深、字段别太多，复杂结构可拆分多次调用。
- **善用枚举**：固定取值用 enum，避免模型自由发挥。
- **给字段加描述**：在 schema 里写清每个字段含义，提升填充准确率。
- **兜底重试**：即便用了约束解码，也保留「解析失败 → 带错误信息重试」的兜底。
- **降低温度**：结构化任务用低温度，减少随机性。
- **注意拒答**：约束解码会「强制」模型一定填出结构，可能掩盖「模型其实不知道」的情况，需在 schema 里留 null/“未知”表达。

## 与 Function Calling 的关系

Function Calling 的「参数」本质就是一次结构化输出——模型按工具的参数 JSON Schema 生成合法参数。所以结构化输出能力是 [Function Calling](/agent/function-calling-mcp) 和 Agent 工具调用可靠性的基础。

## 高频追问

**Q：怎么保证大模型一定输出合法且符合结构的 JSON？** 首选 Schema 约束 / 约束解码（在解码层强制每个 token 合法，几乎 100% 合规）；其次 JSON Mode（只保证合法 JSON、不保证符合你的 schema）；最后才是纯 prompt 要求 + 失败重试。生产环境优先约束解码 + 兜底校验。

**Q：JSON Mode 和 Structured Outputs（Schema 约束）有什么区别？** JSON Mode 只保证输出能被 JSON 解析；Schema 约束（Structured Outputs）进一步保证输出**严格符合你给定的 JSON Schema**（字段、类型、枚举都对）。后者更强、更适合直接对接程序。

**Q：约束解码的原理是什么？** 把目标格式编译成语法/状态机，在生成每一步根据当前状态屏蔽掉所有不合法的候选 token，只从合法集合采样，从而保证整段输出一定符合结构。代价是实现复杂、对某些格式有少量性能开销。

**Q：约束解码有什么副作用？** 它会「强迫」模型填满结构，即使模型其实没有答案，也可能编造字段值（掩盖了拒答/不确定）。应对：在 schema 里显式提供 null 或「未知」选项，并配合校验和必要的二次确认。

**Q：结构化输出和 Function Calling 是什么关系？** Function Calling 生成的工具「参数」就是按参数 schema 的一次结构化输出，所以结构化输出是工具调用可靠性的基础；很多 API 用同一套约束机制同时支撑两者。
